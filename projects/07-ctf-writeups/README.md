# 07 - CTF Writeups

## Цель
Документировать решения CTF-задач для закрепления знаний и демонстрации навыков.

## Формат writeup

Каждый writeup — отдельный файл:

```
07-ctf-writeups/
├── README.md
├── portswigger/
│   ├── sql-injection-lab-01.md
│   ├── xss-reflected-lab-01.md
│   └── ...
├── hackthebox/
│   ├── machine-name.md
│   └── ...
├── tryhackme/
│   └── ...
└── other/
    └── ...
```

## Шаблон writeup

```markdown
# [Название задачи / машины]

**Платформа:** PortSwigger / HTB / THM
**Категория:** Web / Crypto / Forensics / Network
**Сложность:** Easy / Medium / Hard
**Дата решения:** YYYY-MM-DD

## Описание задачи
...

## Разведка (Reconnaissance)
...

## Анализ
Что я заметила, какие гипотезы

## Эксплуатация
Пошаговое описание с командами/запросами

## Что я узнала
- Ключевой takeaway #1
- Ключевой takeaway #2

## Ресурсы
- Полезные ссылки
```

## Рекомендации
- Начни с PortSwigger labs — они структурированы по темам
- Пиши writeup **сразу** после решения, пока помнишь мыслительный процесс
- Не копируй чужие решения — описывай СВОЙ путь, включая тупики
- Объясняй **почему** работает, а не только **как**
